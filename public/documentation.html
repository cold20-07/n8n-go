Introduction

Welcome to N8N Go, the most intuitive way to create, validate, and manage JSON workflows. Our platform combines the power of visual workflow building with real-time validation and seamless integrations.

What is N8N Go?

N8N Go is a comprehensive JSON workflow automation platform that allows you to:

  • Build complex JSON structures visually
  • Validate data in real-time
  • Connect with popular APIs and services
  • Automate data processing workflows

Installation

Getting started with N8N Go is simple. You can use our cloud platform or deploy on-premises.

Cloud Platform

Simply sign up at our website and start building workflows immediately. No installation required.

Self-Hosted

For enterprise users, you can deploy N8N Go on your own infrastructure:

npm install -g @n8n/n8n-go
n8n-go start

Quick Start

Create your first JSON workflow in minutes:

  1. Sign up for a free account
  2. Choose a template or start from scratch
  3. Drag and drop nodes to build your workflow
  4. Configure each node with your data
  5. Test and validate your JSON output
  6. Deploy your workflow

Visual Workflow Builder

Our drag-and-drop interface makes it easy to create complex JSON workflows without writing code. Key features include:

  • Pre-built nodes for common operations
  • Custom node creation
  • Visual connection mapping
  • Real-time preview

Using Templates

API Overview

N8N Go provides a comprehensive REST API for programmatic access to all features.

Base URL

https://your-vercel-domain.vercel.app/api

Generate Workflow

Create a new workflow programmatically:

POST /generate
Content-Type: application/json

{
  "description": "Process customer orders and send notifications",
  "triggerType": "webhook",
  "complexity": "medium"
}

Authentication

Currently, the API is open for public use. Authentication will be added in future versions for premium features.

API Endpoints

Available endpoints:

  • GET /health - Check API status
  • POST /generate - Generate a new workflow
  • GET /api/templates/{id} - Get template details
  • POST /api/templates/suggestions - Get template suggestions

Example Response

{
  "success": true,
  "workflow": {
    "id": "workflow_123",
    "name": "Customer Order Processing",
    "nodes": [...],
    "connections": {...}
  },
  "workflow_name": "Customer Order Processing",
  "node_count": 5,
  "formatted_json": "..."
}
